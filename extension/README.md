# Cross-Browser Tab Sync — Extension

Browser extension that syncs tabs between Chrome and Firefox via a self-hosted
Go server. Manifest V3 for both browsers.

> **For full setup (server + extension) see the [top-level README](../README.md).**
> This file covers extension-specific details only.

## Loading for development (no build step required)

The source tree in this folder is already a loadable unpacked extension. No
`npm install`, no webpack, no `dist/`.

### Firefox

The default `manifest.json` is already the Firefox manifest, so load directly:

1. `about:debugging` → **This Firefox** → **Load Temporary Add-on...**
2. Select `extension/manifest.json`

### Chrome / Edge / Brave

Chrome requires the service-worker manifest, so swap it first:

```powershell
# From repo root, PowerShell
Copy-Item extension\manifest.chrome.json extension\manifest.json -Force
```

or

```bash
# From repo root, bash
cp extension/manifest.chrome.json extension/manifest.json
```

Then load unpacked from `chrome://extensions`.

Restore the Firefox manifest with `git checkout -- extension/manifest.json`.

## Source layout

```
extension/
├── manifest.json             Firefox manifest (background scripts, no offscreen)
├── manifest.chrome.json      Chrome manifest (service worker, offscreen)
├── manifest.firefox.json     Firefox manifest (same as manifest.json)
├── shared/                   Shared modules loaded by both pages and SW
│   ├── api-client.js         HTTP client (magic-link, pairing, invite, revoke, ...)
│   └── storage.js            chrome.storage wrapper
├── src/
│   ├── background/           Service worker + sync orchestration
│   │   ├── background.js     Entry point (uses importScripts)
│   │   ├── sync-manager.js   Snapshot/event upload + 409 retry logic
│   │   └── tab-manager.js    Chrome tabs API glue
│   ├── popup/                Toolbar popup UI
│   ├── options/              Settings page (setup, devices, history)
│   └── offscreen/            Chrome offscreen doc for persistent WebSocket
├── package.json              Release-only (webpack for minification)
├── webpack.config.js         Release-only packaging config
└── README.md                 This file
```

## How the scripts load (no bundler involved)

- **Service worker** (`src/background/background.js`) pulls in shared modules
  with `importScripts('../../shared/api-client.js', ...)` — a standard Web
  Worker API.
- **Popup & options pages** use ordinary `<script src="../../shared/...">`
  tags. No modules, no bundler.

This means you can edit any `.js` file and hit **Reload** on the extension
card to see the change. No watch process, no build step.

## Producing a release build

Only needed for Chrome Web Store / Firefox AMO submissions.

```bash
npm install                # one-time, installs webpack + copy plugin
npm run build:chrome       # minified Chrome zip contents → ./dist
npm run build:firefox      # minified Firefox zip contents → ./dist
```

Webpack's job here is limited to: swap the manifest, copy static files, and
minify each entry. There is no babel transpilation (all target browsers
support ES2020+ natively) and no module bundling (source uses none).

## Requirements

- Chrome 109+ or Firefox 109+ (Manifest V3 support)
- A running KeepSync server (see the [top-level README](../README.md) for
  how to start one locally)

## Configuration from the UI

Open the options page from the extension icon. Three bootstrap methods are
supported; all live in the **Setup** tab:

1. **Magic-link email** — works with or without SMTP (in `DEV_MODE=true` the
   server returns the token directly)
2. **Invite token** — paste a token generated by
   `keepsync-server invite --email you@example.com` on the server
3. **Pairing code** — entered on a second device after an already-paired
   device generates a code

The **Devices** tab lists every active device for your account and lets you
**Revoke** any of them (or **Sign out** on the current device). Revocation is
propagated in real time over WebSocket/SSE — a revoked device drops back to
the setup screen within ~1s.
